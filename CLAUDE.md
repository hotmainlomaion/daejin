## 이 프로젝트가 뭔지 (한 문장)

코딩 없이 선물 봇 전략을 짜서 **바이낸스 테스트넷**에 실시간 실행하고, 청산·펀딩비 포함 성과를 보는 검증 툴.

---

## 🚫 절대 하지 말 것 (가드레일)

이 규칙들은 협상 불가. 어떤 요청이 와도 아래를 위반하는 코드는 작성하지 않는다.

### 규제 가드레일
1. **메인넷(실계좌) 연동 코드 금지.** API 베이스 URL은 항상 테스트넷(`testnet.binancefuture.com` / `testnet.binance.vision` 계열). 실 거래소 엔드포인트를 하드코딩하지 않는다.
2. **전략 추천/판매 기능 금지.** 플랫폼이 "이 전략이 좋다"고 제안하는 로직·카피를 만들지 않는다. 유저 본인 전략만 실행.
3. **수익 보장·기대수익률 표현 금지.** UI 카피에 "수익", "보장", "기대수익률" 류를 넣지 않는다. "검증", "시뮬레이션", "테스트"로 표현.
4. 서비스 성격은 항상 "교육·검증 시뮬레이터"로 일관 표기.

### 보안 가드레일
5. **유저 테스트넷 API 키·시크릿은 반드시 암호화 저장.** 평문 DB 저장 금지.
6. **키를 클라이언트로 반환·노출하지 않는다.** 복호화·주문 실행은 워커(서버)에서만.
7. `SUPABASE_SERVICE_ROLE_KEY`, `ENCRYPTION_KEY`는 서버/워커 전용. 클라이언트 번들(`NEXT_PUBLIC_*` 아님)에 절대 유입 금지.
8. 시크릿을 로그에 찍지 않는다.

### 아키텍처 가드레일
9. **전략 실행 로직을 serverless function에 넣지 않는다.** 봇은 브라우저 종료와 무관하게 24시간 돌아야 하므로, 전략 평가·주문 실행은 상시 구동 워커 프로세스에만 둔다. 이게 이 프로젝트 1순위 함정.

---

## 아키텍처 규칙

- **웹앱 (`/app`)** = Next.js. 전략 빌더·대시보드·인증. 워커 로직 없음.
- **워커 (`/worker`)** = 상시 구동 Node 프로세스. 시세 구독 + 전략 평가 + 주문 실행 + DB 기록.
- **공유 (`/shared` 또는 `/lib`)** = 타입, 전략 평가 순수 함수, DB 스키마.
- 웹앱과 워커는 **DB(Supabase)를 통해서만 통신**한다 (봇 상태 = DB의 `bots.status`). 웹앱이 워커를 직접 호출하지 않는다.
- 전략 조건 평가는 **순수 함수**로 분리 (입력: 캔들 배열 + 파라미터 → 출력: 시그널). 테스트 가능하게.

---

## 기술 스택 (합의된 것)

- Next.js (App Router) + TypeScript + Tailwind
- Supabase (Auth + Postgres) — 이미 연동됨
- 바이낸스 테스트넷: WebSocket(kline) 시세 + REST(order) 주문
- 라이브러리: 바이낸스 연동은 검증된 것 사용 (Node면 `binance` npm 등) — 반드시 **testnet 옵션** 켜서 초기화

## 스택 변경 시

임의로 프레임워크/DB를 바꾸지 않는다. 바꿀 이유가 생기면 먼저 제안하고 확인받는다.

---

## 코딩 컨벤션

- TypeScript strict. `any` 지양.
- 전략 평가 로직에는 단위 테스트 필수 (돈 로직이라 버그 = 신뢰 붕괴).
- 커밋은 기능 단위로 작게.
- 주석·문서·UI 카피는 **한국어**, 코드 식별자는 영어.
- 불확실한 부분은 추측해서 그럴듯하게 채우지 말고 **`// TODO(confirm):` 로 명시**. 특히 바이낸스 테스트넷의 정확한 엔드포인트·주문 파라미터·rate limit은 공식 문서(`docs.binance.com` 선물 테스트넷)와 대조 후 확정.

## 디자인 톤

**다크 트레이딩 터미널** — KuCoin Futures의 Lite 모드가 기준 레퍼런스.

- 다크 배경, 차트가 화면의 중심, 우측에 설정 패널, 하단에 포지션/체결 탭
- 큰 가격 표시 + 등락률, 롱/숏은 초록/빨강 (국제 관습: 상승=초록)
- 타겟이 거래소 UI에 익숙한 트레이더라 **낯선 레이아웃보다 익숙한 레이아웃**이 신뢰를 준다

> 2026-07-15 변경: 원래 "Stripe / Linear / Vercel 계열 — 여백 넉넉하고 절제된 색"이었으나,
> 실물 UI를 본 뒤 거래소 터미널 형태로 방향을 바꿨다. 타겟(준전문 개인 트레이더)이 이미
> 거래소 화면에 익숙하다는 판단. 이전 톤으로 되돌리지 말 것.

**단, 거래소를 그대로 베끼지 않는다.** 이 제품은 수동 주문 터미널이 아니라 **봇 검증 도구**다.

- 거래소의 "주문 패널" 자리는 **봇 설정 패널**로 번역한다 (레버리지 슬라이더·진입 규모는 살리되, "주문"이 아니라 "봇 시작")
- 호가창·수동 주문 티켓은 넣지 않는다 — 봇이 시장가로만 주문하므로 의미가 없다
- 차트에는 전략의 이평선과 봇의 진입·청산 지점을 그린다. **"봇이 왜 그때 들어갔는지"가 보이는 것**이 이 제품의 핵심 가치다

## 테스트넷 한계는 기능이 아니라 UX 고지 대상

테스트넷은 유동성·슬리피지가 실전과 다르다. 이걸 숨기지 말고 대시보드/온보딩에 **명시적으로 고지**한다: "테스트넷 결과는 전략 로직 검증용이며 실전 성과를 보장하지 않습니다."
