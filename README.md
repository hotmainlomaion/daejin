## 무엇인가

비개발자 트레이더가 선물 자동매매 전략을 **실전 자금 리스크 0으로** 검증하게 해주는 서비스.

- 노코드 전략 빌더로 진입/청산 조건 + 레버리지 + 손절 설정
- 바이낸스 테스트넷에 실시간 라우팅 → 청산·펀딩비까지 실전형으로 시뮬레이션
- 성과 대시보드로 결과 확인

핵심 차별점: 남들은 백테스트에서 멈추지만, 우리는 **실시간 시장 위 포워드 테스트**를 노코드로 제공.

---

## 왜 테스트넷인가

바이낸스 선물 테스트넷(`testnet.binancefuture.com`)이 정식 운영 중이라, **체결·청산·펀딩비 엔진을 직접 만들 필요가 없다.** 주문을 실제 테스트넷에 라우팅하면 바이낸스가 실전과 거의 동일하게 처리한다.

- 무료 테스트넷 자금 (가입 시 테스트 USDT 지급)
- 실제 시장 데이터 미러링
- python-binance 등 라이브러리가 testnet 키를 그대로 지원

**한계 (UX에 명시할 것):** 테스트넷은 유동성이 실제와 달라 체결이 비현실적으로 쉽고, 슬리피지가 현실적이지 않으며, 일부 주문 타입이 빠져 있다. → "전략 로직 검증"용이지 "실전 성과 예측"용이 아니라는 걸 유저에게 분명히 알린다.

한국은 무기한 선물의 국내법상 정의가 없는 규제 사각지대. 실질적 단속선은 FIU의 **"한국인 대상 한국어 영업하는 불법 가상자산사업자"** 단속.

- 🚫 실계좌 연동 · "이 전략 쓰면 수익" 마케팅 · 전략 추천/판매 (→ 유사투자자문·유사수신 리스크)

## 기술 스택

| 레이어 | 선택 | 비고 |
|---|---|---|
| 프론트엔드 | Next.js (App Router) + TypeScript + Tailwind | 디자인 톤: Stripe / Linear / Vercel |
| 인증·DB | Supabase (Auth + Postgres) | 이미 연동돼 있음 |
| **전략 실행 엔진** | **상시 구동 Node 워커 서비스** | ⚠️ 서버리스 불가 — 아래 참조 |
| 시세 | 바이낸스 테스트넷 WebSocket (kline stream) | |
| 주문 | 바이낸스 테스트넷 REST API | 유저 testnet 키 사용 |
| 배포 | 웹앱: Vercel · 워커: Railway/Fly.io/VM | 워커는 상시 프로세스 필요 |

### ⚠️ 아키텍처 핵심 제약

봇은 **유저가 브라우저를 꺼도 24시간 돌아야 한다.** 따라서 전략 평가·주문 실행 로직은 Vercel serverless function이 아니라 **별도의 상시 구동 워커 프로세스**에 있어야 한다. 이걸 serverless에 넣으려다 막히는 게 이 프로젝트 최대 함정. 처음부터 워커를 분리해서 시작한다.

## 저장소 구조

```
app/                  Next.js 웹앱 (전략 빌더·대시보드·인증) — 워커 로직 없음
worker/               상시 구동 Node 워커 (시세 구독 + 전략 평가 + 주문 + DB 기록)
shared/               타입 · 전략 평가 순수 함수 · 지표 · 키 암호화
supabase/migrations/  DB 스키마 (Supabase↔GitHub 연동이 적용)
```

웹앱과 워커는 **DB를 통해서만 통신**한다 (봇 상태 = `bots.status`).

## 시작하기

```bash
# 1. 의존성
npm install

# 2. 환경변수 — .env.example을 복사해서 채운다
cp .env.example .env          # 워커용
cp .env.example app/.env.local # 웹앱용 (NEXT_PUBLIC_* + SUPABASE_SERVICE_ROLE_KEY + ENCRYPTION_KEY)
#   ENCRYPTION_KEY는 `openssl rand -hex 32`로 생성. 잃어버리면 저장된 유저 키를 복호화할 수 없다.

# 3. 웹앱
npm run dev

# 4. 워커 (반드시 별도 프로세스)
npm run worker

# 검증
npm test         # 전략·암호화·정밀도 단위 테스트
npm run typecheck
```

바이낸스 테스트넷 키는 **유저가 각자 발급**해서 입력한다 (초기엔 이 방식이 규제·보안상 가장 깔끔). 발급 안내: `testnet.binancefuture.com` → API Key 발급.

### 테스트넷 강제

워커는 기동 시 베이스 URL의 호스트를 화이트리스트와 대조하고, 메인넷 주소면 **주문을 내기 전에 기동을 실패**시킨다 (`worker/src/config.ts`). 가드레일 1을 코드로 강제하는 장치이므로 화이트리스트를 느슨하게 바꾸지 않는다.
